Introduction to etwfe

Extended Two-Way Fixed Effects for Python

Introduction

Standard two-way fixed effects (TWFE) regressions can produce biased estimates when treatment is staggered across units. Wooldridge (2023, 2025) proposed a solution: instead of a single treatment dummy, saturate the model with all cohort×time interactions. The treatment effects are then recovered by aggregating the interaction coefficients via marginal effects.

This extended TWFE (ETWFE) approach is straightforward conceptually but tedious to implement—requiring correct interaction specification, demeaned controls, and proper aggregation. The etwfe package automates these steps.

Dataset

Throughout this tutorial, we use the mpdta dataset—a county-level panel tracking teen employment across 500 US counties from 2003 to 2007. The treatment of interest is minimum wage increases, which were adopted by different counties at different times (2004, 2006, or 2007), making this a classic staggered adoption setting.

python
from etwfe import etwfe
import pyreadr

# Load the minimum wage dataset
url = "https://github.com/bcallaway11/did/raw/master/data/mpdta.rda"
pyreadr.download_file(url, "mpdta.rda")
mpdta = pyreadr.read_r("mpdta.rda")["mpdta"]

# Prepare variables
mpdta["first_treat"] = mpdta["first.treat"].astype(int)
mpdta["year"] = mpdta["year"].astype(int)
mpdta["countyreal"] = mpdta["countyreal"].astype(int)

Basic Usage

Using etwfe is a two-step process:

  1. etwfe() — fits the saturated interaction model
  2. emfx() — aggregates coefficients into interpretable treatment effects

etwfe

The etwfe() function estimates the extended TWFE model. You provide a formula specifying the outcome and controls, along with the time and group (treatment cohort) variables.

python
mod = etwfe(
    fml="lemp ~ lpop",          # outcome ~ controls
    tvar="year",                # time variable
    gvar="first_treat",         # group variable
    data=mpdta,                 # dataset
    vcov={"CRV1": "countyreal"}  # cluster SEs by county
)

All arguments except vcov are required. Clustering standard errors (as shown above) is recommended for panel data.

Under the hood, etwfe() creates a treatment indicator and interacts it with cohort and time variables. The resulting model object is a pyfixest regression, so standard methods like .summary() are available. However, the raw coefficients aren't directly interpretable—they represent complex interaction terms. To get meaningful treatment effects, we use emfx().

emfx

The emfx() method computes marginal effects that aggregate the interaction coefficients into interpretable quantities. The simplest call returns the overall average treatment effect on the treated (ATT):

python
# Simple ATT
mod.emfx()
Average Treatment Effect on the Treated
   _Dtreat  estimate  std.error  conf.low  conf.high
0      1.0 -0.050627   0.012497 -0.075122  -0.026132

The ATT of −0.051 indicates that minimum wage increases reduced teen employment by about 5 percent on average.

The type argument controls how effects are aggregated. Options include "simple" (overall ATT), "event" (by time relative to treatment), "group" (by treatment cohort), and "calendar" (by calendar year). Event studies are particularly useful for examining treatment dynamics:

python
# Event study
mod.emfx(type="event")

Visualization

Plots

The plot() method provides quick visualization of treatment effects:

python
mod.plot(type="event")

By default, only post-treatment effects are shown. With cgroup="notyet" (the default), pre-treatment effects are mechanistically zero since not-yet-treated units serve as controls.

To examine pre-treatment trends (useful for validating the parallel trends assumption), use cgroup="never" to restrict the control group to never-treated units:

python
mod_never = etwfe(
    fml="lemp ~ lpop",
    tvar="year",
    gvar="first_treat",
    data=mpdta,
    cgroup="never",              # use never-treated as control
    vcov={"CRV1": "countyreal"}
)

# Event study with pre-treatment
mod_never.emfx(type="event", post_only=False)

Tables

You can easily compare models side-by-side. Here we compare the never-treated control (with pre-treatment periods) against the not-yet-treated control:

python
import pandas as pd

# Get event study results
es_never = mod_never.emfx(type="event", post_only=False)[["event", "estimate", "std.error"]]
es_notyet = mod.emfx(type="event")[["event", "estimate", "std.error"]]

# Rename columns
es_never.columns = ["event", "est_1", "se_1"]
es_notyet.columns = ["event", "est_2", "se_2"]

# Merge and sort
table = es_never.merge(es_notyet, on="event", how="outer")
table = table.sort_values("event").reset_index(drop=True)

# Get model stats
nobs_1 = mod_never.model_._N
nobs_2 = mod.model_._N
r2_1 = mod_never.model_._r2
r2_2 = mod.model_._r2
rmse_1 = mod_never.model_._rmse
rmse_2 = mod.model_._rmse

# Function to add significance stars
def add_stars(est, se):
    if pd.isna(est) or pd.isna(se) or se == 0:
        return f"{est:.3f}" if pd.notna(est) else ""
    t = abs(est / se)
    if t > 3.291: return f"{est:.3f}***"
    elif t > 2.576: return f"{est:.3f}**"
    elif t > 1.96: return f"{est:.3f}*"
    elif t > 1.645: return f"{est:.3f}+"
    else: return f"{est:.3f}"

# Print formatted table
print("\n" + "="*50)
print("                    Event Study")
print("="*50)
print(f"{'':30} {'(1)':>10} {'(2)':>10}")
print("-"*50)

for _, r in table.iterrows():
    evt = int(r["event"])
    label = f"Years post treatment = {evt}"
    col1 = add_stars(r['est_1'], r['se_1']) if pd.notna(r['est_1']) else ""
    col2 = add_stars(r['est_2'], r['se_2']) if pd.notna(r['est_2']) else ""
    print(f"{label:30} {col1:>10} {col2:>10}")
    se1 = f"({r['se_1']:.3f})" if pd.notna(r["se_1"]) and r["se_1"] != 0 else ""
    se2 = f"({r['se_2']:.3f})" if pd.notna(r["se_2"]) else ""
    print(f"{'':30} {se1:>10} {se2:>10}")

print("-"*50)
print(f"{'Num.Obs.':30} {nobs_1:>10} {nobs_2:>10}")
print(f"{'R2':30} {r2_1:>10.3f} {r2_2:>10.3f}")
print(f"{'RMSE':30} {rmse_1:>10.3f} {rmse_2:>10.3f}")
print(f"{'FE..first.treat':30} {'X':>10} {'X':>10}")
print(f"{'FE..year':30} {'X':>10} {'X':>10}")
print("-"*50)
print("+ p < 0.1, * p < 0.05, ** p < 0.01, *** p < 0.001")
print("Std. errors are clustered at the county level")

Heterogeneous Treatment Effects

Treatment effects often vary across subgroups. The xvar argument enables estimation of heterogeneous effects by interacting treatment with a categorical variable.

Here we examine whether minimum wage effects differ between Great Lakes states and other regions:

python
# Create Great Lakes indicator
gls_fips = [17, 18, 26, 27, 36, 39, 42, 55]
mpdta["gls"] = mpdta["countyreal"].astype(str).str[:2].astype(int).isin(gls_fips)

With xvar specified, emfx(by_xvar=True) returns separate ATTs for each subgroup:

python
hmod = etwfe(
    fml="lemp ~ lpop",
    tvar="year",
    gvar="first_treat",
    data=mpdta,
    cgroup="never",
    xvar="gls",                  # heterogeneous TEs by gls
    vcov={"CRV1": "countyreal"}
)

# ATT by subgroup
hmod.emfx(by_xvar=True)

1 -0.039396   0.024296 -0.087016   0.008223      1.0   True

The event study visualization shows how effects evolve differently across groups:

Non-Linear Models

ETWFE extends naturally to non-linear models via the family argument. This is useful for count outcomes (Poisson), binary outcomes (logit/probit), and other GLM families.

Here's the same analysis using Poisson regression on employment counts:

python
import numpy as np

# Create employment count (exponentiate log employment)
mpdta["emp"] = np.exp(mpdta["lemp"])

# Fit Poisson ETWFE model
mod_pois = etwfe(
    fml="emp ~ lpop",
    tvar="year",
    gvar="first_treat",
    data=mpdta,
    vcov={"CRV1": "countyreal"},
    family="poisson"
)

mod_pois.emfx(type="event")

Performance Tips

The etwfe() step is fast. The emfx() step can be slower for large datasets (100k+ rows) due to the marginal effects computation.

Two strategies can speed things up:

  1. Skip standard errors: emfx(..., vcov=False) is much faster and useful for initial exploration.
  2. Compress the data: emfx(..., compress=True) aggregates data before computation, trading a small accuracy loss for significant speed gains.

A practical workflow for large datasets:

  1. Run emfx(vcov=False) to check point estimates quickly
  2. Run emfx(vcov=False, compress=True) and compare
  3. If estimates match closely, use emfx(compress=True) for approximate standard errors
python
# Step 1: Quick estimates without SEs
mod.emfx(type="event", vcov=False)

# Step 2: With compression (faster for large data)
mod.emfx(type="event", vcov=False, compress=True)

# Step 3: If results match, get approx. SEs with compression
mod.emfx(type="event", compress=True)

Compare the original and compressed results:

python
import pandas as pd

# Get event study results - original and compressed
es_original = mod.emfx(type="event")[["event", "estimate", "std.error"]]
es_compressed = mod.emfx(type="event", compress=True)[["event", "estimate", "std.error"]]

# Rename columns
es_original.columns = ["event", "est_orig", "se_orig"]
es_compressed.columns = ["event", "est_comp", "se_comp"]

# Merge
table = es_original.merge(es_compressed, on="event", how="outer")
table = table.sort_values("event").reset_index(drop=True)

# Get model stats
nobs = mod.model_._N
r2 = mod.model_._r2

# Print formatted table
print("\n" + " "*20 + "Event study")
print(" "*30 + "Original  Compressed")
print("-"*55)

for _, r in table.iterrows():

Function Reference

etwfe()

Fit an Extended Two-Way Fixed Effects model.

ParameterTypeDescription
fmlstrFormula: "outcome ~ controls" or "outcome ~ 0"
tvarstrTime period variable name
gvarstrTreatment cohort variable (first treatment period; 0 for never-treated)
dataDataFramePanel dataset
ivarstrUnit identifier (optional). Defaults to group-level fixed effects.
xvarstrHeterogeneous effects variable (optional)
trefintReference time period (optional). Defaults to minimum.
grefintReference group (optional). Defaults to 0 (never-treated).
cgroup"notyet" | "never"Control group: not-yet-treated or never-treated
festrFixed effects specification: "vs" (varying slopes, default), "feo", or "none"
familystrGLM family: "poisson", "logit", "probit", etc.
vcovstr | dictVariance estimator, e.g., {"CRV1": "cluster_var"}

model.emfx()

Compute marginal effects (ATT estimates).

ParameterTypeDescription
typestr"simple", "event", "group", or "calendar"
by_xvarboolCompute effects by xvar levels (default False)
post_onlyboolOnly post-treatment periods (default True)
compressboolCompress data for faster computation (default False)
vcovboolCompute standard errors (default True; False to skip)
predictstrPrediction type (default "response")

model.plot()

Visualize marginal effects.

ParameterTypeDescription
typestr"event", "group", or "calendar"
stylestr"errorbar" (default) or "ribbon"
post_onlyboolOnly post-treatment periods
by_xvarboolPlot by xvar groups
figsizetupleFigure dimensions
colorstrPlot color (single group)
colorslistColors for multiple groups
